## üöÄ About The Project

**CodeMentorAI** isn't just another code completion tool. It's designed to be a **teacher**. Instead of giving you the answers outright, it guides you to the solution, helping you understand the underlying concepts and build real problem-solving skills.

Whether you're struggling with a syntax error in Python, a logic bug in C++, or just starting your journey with Java, CodeMentorAI is here to help you **learn, not just copy**.

### ‚ú® Key Features

*   **üß† Learning-First Pedagogy**: The AI is prompted to act as a mentor, offering hints and explanations rather than direct solutions.
*   **üí¨ Interactive Chat Interface**: Have a natural conversation about your code. Ask follow-up questions and get instant feedback.
*   **üíª Professional Code Editor**: Integrated Monaco Editor (VS Code engine) with syntax highlighting for Python, C, C++, and Java.
*   **üì∑ Visual Problem Helper**: Upload screenshots of coding problems or errors. The AI extracts text (OCR) and provides targeted help.
*   **üîä Voice Explanations**: Audio support for explanations, perfect for auditory learners.
*   **üìà Progress Tracking**: Keep track of your learning journey with session timers and question metrics.
*   **üåì Dark/Light Mode**: Beautiful, responsive UI that adapts to your preference.

## üèÅ Getting Started

Follow these instructions to set up the project locally on your machine.

### Prerequisites

*   **Node.js** (v18 or higher)
*   **npm** (v9 or higher)
*   **Google Gemini API Key**

### üì• Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/AasishDairelSahayaGrinspan/grasp-hackathon.git
    cd grasp-hackathon
    ```

2.  **Setup Backend**
    ```bash
    cd learning-tutor-backend
    npm install
    # Create a .env file based on .env.example
    echo "GEMINI_API_KEY=your_api_key_here" > .env
    npm start
    ```

3.  **Setup Frontend** (Open a new terminal)
    ```bash
    cd learning-tutor-frontend
    npm install
    npm run dev
    ```

Your app should now be running at `http://localhost:5173`!
